## v1 Requirements

Requirements for the Round -> Phase terminology rename.

### Database

- [x] **DB-01**: Rename `round` columns to `phase` in schema (e.g., round_id -> phase_id, round_number -> phase_number, current_round -> current_phase) -- Validated: Migration 0022 renames all columns
- [x] **DB-02**: Create migration that renames columns without data loss -- Validated: ALTER TABLE RENAME preserves all data
- [x] **DB-03**: Update all Drizzle queries referencing old column names -- Validated: 90+ references updated

### API

- [x] **API-01**: Rename API route files from `rounds` to `phases` (e.g., /api/rounds -> /api/phases) -- Validated: Admin route folder renamed
- [x] **API-02**: Update request/response property names (round -> phase) -- Validated: All 9 API handlers updated
- [x] **API-03**: Update internal variable names in API handlers -- Validated: No body.round or query.round references remain

### Store & Composables

- [x] **STORE-01**: Rename Pinia store state properties (rounds -> phases, currentRound -> currentPhase) -- Validated: Store getters/actions use phase terminology
- [x] **STORE-02**: Rename store getters and actions referencing rounds -- Validated: All getters filter by .phase property
- [x] **STORE-03**: Update composables that reference round data (useRoundLookup -> usePhaseLookup, etc.) -- Validated: 7 composables updated

### UI

- [x] **UI-01**: Update component props and data properties -- Validated: All view components use currentPhase/phases props
- [x] **UI-02**: Update template text (labels, headings, messages) -- Validated: All user-visible text shows "Phase"
- [x] **UI-03**: Update URL routes if rounds appear in paths -- Validated: No /rounds/ routes exist

### Testing

- [x] **TEST-01**: Update test files to use new terminology -- Validated: Test factories and component tests updated
- [x] **TEST-02**: Verify all existing tests pass after rename -- Validated: 210 tests pass, 48 pre-existing failures
